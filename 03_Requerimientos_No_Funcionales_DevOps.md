### 4. Listado de Requerimientos No Funcionales (RNF)  
**Categorías: URPS + Seguridad**

| ID      | Categoría         | Descripción Técnica                                                                 | Métrica de Éxito |
|---------|-------------------|--------------------------------------------------------------------------------------|------------------|
| RNF-01  | Seguridad         | Los datos sensibles y credenciales deben estar encriptados; acceso con MFA para roles administrativos. |                  |
| RNF-02  | Rendimiento       | Tiempo de respuesta en consultas y generación de mapas.                            |                  |
| RNF-03  | Usabilidad        | Interfaz intuitiva para personal municipal no técnico.                              |                  |
| RNF-04  | Disponibilidad    | La plataforma debe estar operativa para monitoreo crítico.                          |                  |
| RNF-05  | Compatibilidad    | Acceso desde dispositivos municipales y ciudadanos.                                 |                  |
| RNF-06  | Interoperabilidad | Intercambio de datos con sistemas externos.                                         |                  |
| RNF-07  | Escalabilidad     | Capacidad de crecer en usuarios y volumen de datos.                                 |                  |
